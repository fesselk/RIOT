    1 # Removed Features and Modules
    2 
    3 This document contains a listing of all features and modules that were removed
    4 from RIOT at some point in time, mostly due to missing maintenance. The purpose
    5 of this list is to have a reference point for reintegrating those features in
    6 the future, if new interest arises or a new maintainer is found.
    7 
    8 This list is **not** supposed to contain a change log of all the things that are
    9 being removed from RIOT, but should contain only full modules that are
    10 removed without any alternative being merged at the same time. For example
    11 things like the first `netdev` should not be put in this list, as a successor
    12 (former `netdev2`) has been merged.
    13 
    14 
    15 ## How to read this list
    16 
    17 For each high-level feature removed there should be one entry in this list. The
    18 entries should comply to the following template:
    19 
    20 ~~~~~~~~~~~~~~~~~~~ {.md}
    21 ### path/feature_name [HASH of removal commit]
    22 Author(s):
    23 - author 1 <foo.bar@abc.com>
    24 - author 2 <a.b@c.net>
    25 
    26 Reason for removal:
    27 - give a short and comprehensive reasoning why this feature was removed
    28 - typical reasons are:
    29 - feature is not maintained anymore
    30 - lack of hardware so feature can not be tested (anymore)
    31 - feature was never used and there is no reason for keeping it
    32 ~~~~~~~~~~~~~~~~~~~
    33 
    34 By putting the name of the removed feature and the commit hash in the same line,
    35 one can very quickly find the commit using `git grep`.
    36 
    37 Listing the authors of the removed code is done for appreciation of their work.
    38 This way, their names are never removed from the RIOT repository.
